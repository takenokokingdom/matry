# Matry 仕様書

## サービス概要

**サービス名**：Matry

**コンセプト**：プログラミング知識ゼロでも、AIと会話しながら自分だけのスマホアプリが作れて、ちゃんと使えるサービス。アプリの中でアプリを作る入れ子構造からMatryoshkaを連想して命名。

**ターゲット**：非エンジニア。自分用アプリを持ちたいけど作れない人。

**差別化**：AIがフルコード生成するため自由度が段違い。テンプレ感ゼロ、普通のアプリと遜色ない体験。

## 技術スタック

| 項目 | 技術 |
|------|------|
| モバイル | React Native（Expo） |
| バックエンド | Next.js API Routes |
| ホスティング | Vercel |
| DB・認証 | Supabase |
| 実行環境 | WebView + Babel standalone |
| AI | Claude API / claude-sonnet-4-6（ストリーミング） |
| エージェント | 自前実装（フレームワーク不使用） |

---

## システム構成

```
React Native アプリ（Expo）
  ホーム画面・会話UI・プレビュー（WebView + Babel standalone）
        ↕
Next.js API Routes（Vercel）
  自前エージェントループ。Claude API呼び出し・tool_use制御・Supabase操作
        ↕                    ↕
Claude API              Supabase
会話・ヒアリング         アプリ保存・ユーザーDB
コード生成・tool_use     共有データ管理・認証
```

---

## AIエージェント構成（コード生成ループ）

フレームワーク不使用。Next.js API Routes上で自前ループを実装。

```
① Claude API でReactコード生成（ストリーミング）
        ↓
② サンドボックスで検証
   - Babelトランスパイルエラー
   - 構文エラー・import漏れ
   - 使用不可ライブラリの参照
   - ランタイムエラー
        ↓ エラーあり（最大3回まで）
③ エラー内容をClaudeにフィードバック → 修正
        ↓ エラーなし
④ プレビュー表示
```

---

## アプリ作成フロー

1. **会話開始** — 画面下のテキストエリアに話しかける。アプリ一覧がふわっと消えて会話モードへ
2. **AIがヒアリング** — 質問しながら要件を一緒に詰める。会話履歴がそのまま仕様書になる
3. **生成** — 「まずは作ってみましょう」でReactコンポーネントをストリーミング生成。エージェントがエラー検証ループを回す
4. **フルスクリーンプレビュー** — 丸ボタン長押しでメニュー展開（保存・共有・フィードバック・停止）
5. **名前を決める** — 自分で入力 or AIが候補を提示
6. **ホーム画面に配置して完成** — 会話継続・別アプリ作成・リセットも可能

## UIデザイン

- ダークテーマベース、シックでミニマル
- ホーム画面：アプリアイコン一覧 + 画面下部チャットバー
- 会話中：アイコン一覧がフェードアウト、チャットUIに切り替わり
- プレビュー：フルスクリーン没入型、フローティング丸ボタン長押しでメニュー展開（保存・共有・フィードバック・停止）

## アプリストア

### 概要
- ユーザーが作ったアプリを公開投稿できる
- 一覧・検索・カテゴリ分けで探せる
- 他人のアプリをそのまま自分のホームに追加できる

### 更新管理
- 元アプリが更新されると、追加済みユーザーに「更新があります」と通知
- リリースノート確認 → 規約表示 → 「更新する」を明示的に押さない限り更新されない
- 強制更新なし（第三者製アプリのため）

### レビュー機能
- 星5段階評価
- コメント投稿
- コメントへの返信（スレッド形式）

### フォーク機能
- 他人のアプリをフォークして自分のものにできる
- フォーク元の情報が常に紐づいて表示される
- フォーク版を再公開する場合、派生元が明記される（「based on ××」）

### AI審査
- 公開前にコードをAIが自動スキャン（手動審査なし）
- チェック項目：個人情報の窃取・外部への不審な通信・危険なAPI呼び出し・マルウェア的な挙動
- 審査通過したものだけストアに掲載
- 通報機能・悪用対策は今後検討

## 機能一覧

### フェーズ1（コア）
- 会話でアプリ作成
- ストリーミング生成
- エラー検証ループ（自前エージェント）
- フルスクリーンプレビュー
- アプリ保存・一覧
- ホーム画面配置（アイコン付き）
- 他人へ共有
- 自分専用DB
- AI連携（Claude API）
- アプリストア（公開・検索・追加・レビュー・フォーク）
- AI自動審査

### フェーズ2（拡張）
- マイク・録音
- カメラ・録画
- プッシュ通知
- ネイティブブリッジ（スマホ機能との連携）
- データ共有（家族アルバム等）
- フローティング常駐（Androidのみ）
- 通報・悪用対策

---

## DB要件（設計は別途）
- アプリのバージョン履歴を管理できること
- 会話履歴は1メッセージ1レコードで持つこと
- ユーザーごとにデータが分離されること
- アプリの公開・共有・フォーク関係を管理できること
- レビュー・コメント・返信のスレッド構造を持てること
- フォーク元との親子関係を管理できること
- AI審査結果を記録できること
